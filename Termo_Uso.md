# 📄 Termos de Uso - Smarko (Autenticação Defensiva)
 
Este documento define as regras, obrigações e responsabilidades para o uso do sistema **Smarko**, uma plataforma de automação comercial com arquitetura focada em segurança defensiva.
 
---
 
## 1. Aceitação e Objeto
Ao utilizar o Smarko, o usuário concorda integralmente com as disposições aqui estabelecidas. O sistema provê ferramentas de gestão comercial e camadas de segurança avançadas, incluindo criptografia de credenciais e autenticação multifator (MFA).
 
## 2. Responsabilidades do Usuário
Dado que o Smarko é um sistema focado em segurança defensiva, o usuário compromete-se a:
* **Integridade das Credenciais:** Manter a guarda segura de sua senha e do acesso ao e-mail cadastrado para o Segundo Fator de Autenticação (2FA).
* **Uso Ético:** Não utilizar o sistema para processar dados obtidos de forma ilícita ou para realizar testes de vulnerabilidade não autorizados em ambientes de terceiros.
* **Dados de Cadastro:** Fornecer informações verídicas, especialmente o endereço de e-mail, que é o pilar da validação de identidade do sistema.
 
## 3. Segurança e Proteção Defensiva
O Smarko utiliza tecnologias de ponta para proteção de dados sensíveis:
* **Habilitação de Segurança:** O usuário está ciente de que tentativas consecutivas de login malsucedidas ativarão o mecanismo de *Rate Limiting*, podendo resultar no bloqueio temporário do IP ou da conta para proteção contra ataques de força bruta.
* **Criptografia:** Todas as senhas são processadas via algoritmo **BCrypt**, garantindo que nem mesmo os administradores do banco de dados tenham acesso à senha em texto claro.
 
## 4. Disponibilidade e Versão de Desenvolvimento
O sistema encontra-se em fase de **Desenvolvimento Ativo**.
* Embora o Smarko seja projetado para suportar dados sensíveis com segurança, o usuário reconhece que instabilidades técnicas podem ocorrer durante esta fase de implementação.
* Recomenda-se a manutenção de backups externos dos dados operacionais inseridos no sistema.
 
## 5. Limitação de Responsabilidade
* Nenhum sistema de segurança é 100% invulnerável.
* Os autores não serão responsabilizados por invasões decorrentes de falhas de segurança no dispositivo do próprio usuário ou por negligência no gerenciamento das credenciais de acesso.
 
## 6. Propriedade Intelectual
O código-fonte, a arquitetura de segurança e o design do Smarko são protegidos por direitos autorais. O uso comercial ou a redistribuição do software depende de autorização expressa dos autores ou do cumprimento da licença específica anexada ao projeto.
 
## 7. Foro e Alterações
Estes termos podem ser atualizados para refletir novas funcionalidades de segurança. O uso contínuo do sistema após alterações implica na aceitação dos novos termos.
